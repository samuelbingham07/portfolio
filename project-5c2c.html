5C2C

5C2C is a campus marketplace built for the Claremont College Consortium — a platform where students can formalize their side hustles, post services, and get paid for what they do best. Nail art, graphic design, music production, hair styling, henna, rides — all of it in one place, visible to the exact community that wants it.

The Problem

The 5Cs are full of talented students running real services, but almost all of it lives on scattered Instagram accounts and word-of-mouth group chats. There's no central place to discover what's available, no structured way to book or pay, and no visibility for the students doing the work. Buyers have to know the right people. Sellers have no way to grow beyond their existing social circle. 5C2C fixes that by giving student freelancers a proper storefront and buyers a proper marketplace — built by students, for students, right on campus.

What It Does

The platform has two sides. As a service provider, you create a shop: upload photos of your work, set your prices, write your story, and manage your listings and availability. As a buyer, you browse a card-based feed of services (Instagram-style), filter by category, view detailed profiles with portfolios and ratings, book appointments, and message sellers directly through real-time chat. The full feature set includes:

  • Browse and search services with category filters (nails, design, rides, etc.)
  • Seller profile pages with portfolio photos, pricing, and average ratings
  • "Create Listing" flow with image upload to Supabase Storage
  • Booking system: pick a service, select a time, confirm details
  • Reviews and ratings displayed on listings and profiles
  • Real-time messaging via Socket.io with unread message indicators
  • .edu email verification so the marketplace stays within the 5C community

My Role — All Frontend

I owned the entire frontend of the application. The project runs on a React.js + Tailwind CSS stack, with a Node.js/Express backend and Supabase handling auth, database (PostgreSQL), and file storage. My work covered every user-facing screen and component:

  • Built the homepage, browse/search page, listing cards, and individual listing detail pages
  • Implemented category filters, search bar, and the Instagram-style grid layout for service browsing
  • Built the seller profile pages (portfolio, bio, ratings display) and the buyer-facing booking flow
  • Created the "Create Listing" and "My Listings" management pages, including image upload UI connected to Supabase Storage
  • Built the chat interface — message list, input, send button, inbox/conversations list, and unread indicators — integrated with the Socket.io real-time layer from the backend
  • Implemented protected routes, the navigation bar, and the overall app structure
  • Connected all frontend components to the backend REST APIs throughout the build

The project is co-managed by Arushi Bhandari and Vanisha Kheterpal (both Harvey Mudd '29), and built by a team of 5–7 students across a 10-week sprint.

Through this project I've learned and practiced:

  • React.js component architecture and state management across a multi-page application
  • Tailwind CSS for consistent, responsive UI design at scale
  • Connecting a React frontend to a Node.js/Express REST API
  • Supabase Storage integration for user-uploaded images
  • Real-time UI with Socket.io — building a live chat interface that updates without refreshing
  • Marketplace UI/UX patterns: listing cards, filters, booking flows, and profile pages
  • Collaborating on a shared codebase via GitHub with a cross-functional student team
