Clicktionary

Clicktionary is a Chrome extension that lets you highlight any word on any webpage and instantly get a translation in 10 languages — then save it to your personal word bank and practice later with flashcards and multiple choice. It lives entirely in the browser: no new tab, no copy-pasting, no losing your place in the article you were reading. Coming to the Chrome Store soon.

The Problem

Learning a language by reading real content — news, books, blogs — is one of the most effective methods, but it comes with constant friction. Every time you hit an unfamiliar word, you have to switch tabs, open a translator, copy the word in, read the definition, then find your way back. You lose your reading flow, and because there's no structured follow-up, you forget the word within hours. Clicktionary removes that friction entirely: the translation comes to you, inline, in under two seconds, and the words you care about go straight into a study queue.

How It Works

The extension injects a content script into every webpage. When you highlight text, a tooltip appears with the translation fetched from Lingva Translate — a free, open-source Google Translate-quality API — with MyMemory as a fallback. From the tooltip you can add the word to your word bank with one click, or type your own definition if the automatic lookup misses. The full pipeline:

  • Multiple Lingva Translate instances are queried in parallel via Promise.any() with a 2-second timeout — whichever instance responds first wins, making translations feel instant even when one server is slow.
  • On every page load, the extension silently prefetches the 10 hardest words on the page (longest unique words, which are statistically rarer vocabulary) so that if you highlight one of them, the result is already cached.
  • An in-memory translation cache means repeat lookups on the same word are instant, with no extra network calls.
  • Words you save go to a Supabase database (when signed in) with row-level security, or fall back to chrome.storage.local so the extension works fully offline.
  • A companion GitHub Pages site hosts the full word bank and practice interface — flashcards, multiple choice, and a 0–5 mastery strength tracker synced to Supabase.
  • A bridge content script (bridge.js) is injected into the GitHub Pages site to pass the stored Supabase session over, so you stay logged in across extension and website without re-authenticating.

Technical Highlights

  • Built on Manifest V3 — the current Chrome extension standard — with a background service worker handling auth, Supabase sync, and word bank operations.
  • Google OAuth implemented via chrome.identity.launchWebAuthFlow, with the extension's ID locked to a stable value via an RSA key in the manifest so the OAuth redirect URI never changes.
  • Duplicate detection on word saves, custom definition fallback if translation fails, and an on/off toggle that persists across sessions.
  • Supports 10 languages: Spanish, French, German, Italian, Portuguese, Japanese, Chinese, Korean, Arabic, and Russian — with correct Unicode range detection to decide whether highlighted text could belong to the selected language before showing a tooltip.

Through this project I've learned and practiced:

  • Chrome Extension architecture (Manifest V3): content scripts, service workers, popup, and cross-context messaging
  • Browser extension auth flows — Google OAuth via chrome.identity, session bridging between extension and hosted website
  • Translation API design — parallel instance racing, timeout handling, fallback chaining, and in-memory caching
  • Supabase: row-level security, REST API, and real-time session management
  • Speculative prefetching as a UX performance technique
  • Offline-first design with chrome.storage.local as a graceful fallback
  • Product scoping for a zero-friction browser tool — every interaction had to feel instant or it defeats the purpose
