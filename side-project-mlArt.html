Machine Learning Art

This came out of a course called ML for Artists, which treated machine learning as a creative medium rather than an engineering tool. The series I made was about video game addiction and the way it can negatively transform people and how they see themselves, the psychological phenomenon where someone becomes so invested in a game that the boundary between themselves and their character starts to dissolve. The video above is the only surviving piece; the rest of the series and the other projects from the course were lost, which is a bummer, but at least this one made it.

The pipeline was built around DeForum with prompt interpolation as the core, which handles the actual morphing by interpolating between prompts across frames, going from "human" to "game character" gradually. On top of that, ControlNet preserved the figure's structure and pose as the style transformed around it, LoRA models kept the character consistent across frames, and IPAdapter fed in reference images to guide the look. AnimateDiff smoothed the animation between frames, and Deep Dream was layered in as a post-process for the distortion effect during the transformation. Stable Diffusion was the base throughout.

I have mixed feelings about AI-generated art as a category, but using these tools to say something specific about a real psychological experience felt different from using them as a shortcut. Making something with them rather than just generating for its own sake felt like a more honest use of them, and honestly the results were pretty cool!

Through this project I've learned and practiced:

  • Stable Diffusion and LoRA model fine-tuning for consistent character and style generation
  • AnimateDiff and DeForum for AI video generation and frame interpolation
  • ControlNet and IPAdapter for guided image generation with reference inputs
  • Deep Dream for psychedelic, recursive visual transformation
  • t-SNE for visualizing high-dimensional data embeddings
  • Using ML tools with a deliberate creative concept rather than generating for its own sake
